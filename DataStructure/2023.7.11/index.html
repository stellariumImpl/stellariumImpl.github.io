Linked List

Recap

What are the differences between head pointer and head node?

  • head pointer: which is a necessary element of the linked list. Pointing to the first node of a linked list, if there is a head node, this pointer points to the head node. So regardless of whether the linked list is empty or not, the header pointer is not empty. Moreover, it has the role of identification. Therefore, the head pointer is often used as the name of the linked list.
  • head node: refers to a node that is established for the purpose of unified and convenient operations. It is placed before the first element node in a linked list, and its data field typically has no meaningful value although it can also be used to store the length of the linked list. With a head node, the operations of inserting a node before the first element node and deleting the first node are unified with the operations performed on other nodes. This is because the head node acts as a placeholder and provides a consistent point of reference for manipulating the linked list, regardless of the specific operation being performed. However, a head node is not necessary.

LinkedList is the pointer pointing to struct Node. LinkedList L; declares a linked list and initializes its pointer L to NULL. Node *p;, declares a pointer p that is used for operations within the linked list, which is used to traverse the linked list and access the elements of each node.

Access an element from a linked list. Insert an element into a linked list. Delete an element from a linked list.

Linked List has an advantage over Sequential List in terms of time complexity for deleting and inserting elements.

  • Deletion in Linked List takes constant time complexity of O(1) as it only requires modifying the pointers of nodes. No other elements need to be moved. In contrast, deletion in Sequential List requires moving elements in the array, resulting in a time complexity of O(n) where n is the length of the Sequential List. Elements need to be rearranged after deletion.

  • Insertion in Linked List also takes constant time complexity of O(1) as it only requires modifying the pointers of nodes. No other elements need to be moved. On the other hand, insertion in Sequential List requires shifting all elements after the insertion point, resulting in a time complexity of O(n) where n is the length of the Sequential List. Elements need to be rearranged after insertion.

In summary, Linked List offers constant time complexity for deletion and insertion operations O(1), while Sequential List requires moving elements and has a time complexity of O(n). Therefore, Linked List has higher efficiency and advantages in scenarios involving frequent deletion and insertion operations. However, it should be noted that Linked List lacks the ability to access elements quickly and requires traversing the list to find specific positions.
