Sequential List

Recap

Linear list has two physical storage structure: Sequential and Linked. The former one uses continuous address to store elements one by one.

How to think about GetElem(), this is equivalent to returning the value of the ith position element in the linear list L, just returning the value of index i-1 in the array.

Technically 0 1 2 … i-1 … n-1

What about funtion ListInsert()? Insert new element e into ith position of list L. First we should think about if array defined capacity is not enough for the sequential list and if index i is not between range, so the volume of it should be expanded. Delete ith position element, store it with e, length of sqlist - 1.

Locate element by its value.

To sum up, from its struct and GetElem(), we can know that the sequential structure of a linear list allows for random access, as the elements are stored consecutively in memory. With the starting address and the size of the data elements, the ith element can be immediately located. This is the manifestation of the random access characteristic, and therefore, the time complexity of a sequential list is O(1).

For value-based search, deletion, and insertion operations, the time complexity is O(1) in the best-case scenario and O(n) in the worst-case scenario. Therefore, the average-case time complexity is O(n).

The sequential structure of a linear list is referred to as random access because it allows direct access to elements based on their physical storage location. In a linear list, the elements are sequentially stored in consecutive addresses in memory or storage media. Due to the contiguous storage property, the sequential structure enables fast access (or modification) to specific positions of elements using indices or offsets. For example, in an array that represents a linear list, if we want to access the element at the i-th position, we can easily calculate the address of that element and directly access it without having to traverse the entire structure. In contrast, other non-sequential structures (such as linked lists) do not possess the random access characteristic. In a linked list, elements are not necessarily stored in the order of their physical addresses. Each element only holds the address of the next element, requiring us to start from the head and traverse the linked list to find a specific position’s element.
