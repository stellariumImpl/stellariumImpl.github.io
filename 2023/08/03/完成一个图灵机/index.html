在C和Python中，数组确实只能通过负索引进行访问，并且无法直接访问超出数组长度的内存位置。这是因为数组在内存中是一块连续的内存空间，而且数组的索引是基于内存地址的偏移量。但是也有差别，C语言的数组索引可以是负值，只不过并不是像python一样指向的是从最后数的第几个元素，这里有一个恒等的关系a[i] = *(a+i)，所以只要指向的这个地址里面有内容就可以使用负值索引。

对于两者共同而言，超出数组长度的索引代表的内存位置是属于其他变量、数据结构或是未分配的内存区域。因此不允许直接访问这些未分配的内存区域。

A list of 5 strings is not the best representation of an instruction. Recall that since a TM is deterministic, no two distinct instructions share the same first two elements. A good way to put it is that the program of a TM is a function that maps a pair of the form (state,symbol)(\mathit{state}, \mathit{symbol})(state,symbol) to a triple of the form (new_state,new_symbol,direction)(\mathit{new\_state}, \mathit{new\_symbol}, \mathit{direction})(new_state,new_symbol,direction): (state,symbol)(\mathit{state}, \mathit{symbol})(state,symbol) represents a possible configuration (what is possibly the current state and the symbol stored in the cell that the head of the TM is currently positioned under), while (new_state,new_symbol,direction)(\mathit{new\_state}, \mathit{new\_symbol}, \mathit{direction})(new_state,new_symbol,direction) represents what to do in case the computation is at a stage when that possible configuration happens to be the current one. Start from the Turning Machine

At any stage of the computation, including before it starts and when it ends, if it ever comes to an end, the TM is in one of a finite number of states. The program of a
TM consists of a finite set of instructions, each instruction being a quintuple of the form (state, symbol, new_state, new_symbol, direction)

Suppose that there are two possible states, green and blue. Also suppose that the initial state, that is, the state of the TM before computation starts, is green. If the program of the TM has no instruction whose first two members are green and 1, then computation stops. One the other hand, if the program has one such instruction.
