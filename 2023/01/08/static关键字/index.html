static关键字

被 static 修饰的变量属于类变量，可以通过类名.变量名直接引用，而不需要 new 出一个类来，同理对于类函数也是如此，他们统一属于类的静态资源，被类实例共享，牵一发而动全身。

声明为 static 的对象将分配到静态存储区中，并且一直作用到程序结束。即使函数被多次调用，静态成员变量的空间也只被分配一次，被所有对象共用，并且前一次调用的变量值会被带到下一次函数调用中。有助于 C/C++ 中实现协程或其他需要存储函数先前状态的应用。

调用 static 函数的方式有两种，一是通过 object 调用，二是通过 class name 调用。

类名来调用类的非静态成员函数

Point::init(); 报错 a nonstatic member reference must be relative to a specific object ，得出结论不能通过类名来调用类的非静态成员函数，但是可以通过类对象调用静态或非静态成员函数。

静态数据成员既可以通过对象名引用，也可以通过类名引用。

类的静态成员函数使用类的非静态成员

m_x 报错 a nonstatic member reference must be relative to a specific object ，这是因为静态成员变量属于整个类，在类实例化对象之前就已经分配空间了，而类的非静态成员必须在类实例化对象后才有内存空间，所以这个调用就出错了，就好比没有声明一个变量却提前使用它一样。

类的非静态成员函数使用类的静态成员

编译通过，因为静态成员属于类本身，在类对象产生之前，类初始化的时候加载的时候，就已经存在了，而非静态资源是类实例化对象的时候加载的，所以在非静态成员函数中是可以调用静态成员的。

类的静态成员变量未进行初始化

编译通过，链接报错 unresolved external symbol "private: static int Point::m_nPointCount" (?m_nPointCount@Point@@0HA) ，这是因为类的静态成员变量在使用前必须先初始化，在 main 函数前加上 int Point::m_nPointCount = 0; ，再进行编译链接就可以正常输出了。静态数据成员可以被初始化，但是只能在 class 的外头进行初始化，给不给赋初值其实都可以，如果不给则编译器会自动为其初始化为 0。

相比于一般的成员函数

静态函数相比于一般成员函数，差别在于静态函数没有 this 指针。

可见他无法访问自己类的非静态成员，如果要处理数据的话，那只能处理静态的数据。
