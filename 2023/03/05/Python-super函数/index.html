Python super函数

注： 本文是对Python 多继承中的补充

在开始前，先来看一个例子

Root中的f函数会被执行吗

从C开始，为什么分别调用了其自身、A、B中的f函数，却没有调用Root中的f函数？

首先可以确定，Root中的f函数即时被调用，也是最后执行；其次注意在class B中，也没有对父类的调用（super），这导致在调用完B中的f函数后就结束了。

如果B没有Root这个父类，执行流程为C->A->Root。

如果B中添加了super函数，执行流程为C->A->B->Root，但是注意，A的super调用仍然是B，而非父类Root。

所以super是什么？为什么super函数时而调用兄弟，时而调用父类？

super函数返回什么

super函数的返回值是super类的一个实例，称为proxy object，存储当前运行的类以及当前正在运行的对象，当调用super实例方法时，super根据MRO来查找下一个类，并且调用那个类中的方法。

super函数如何调用

super函数事实上不是调用父类中的函数，而是调用MRO顺序中下一个类的函数。

如何确保父类中的方法调用

  • 规则1：做一个基类，所有内容继承自该类，或者也可以都继承自内置的set或dict等类。
  • 规则2：如果某类中某一个方法中使用super函数，那么你应该在所有该方法中使用super函数，可能除了Root之外。

这样最终就能使得父类中的方法被调用，父类可能不是下一级，但无论是什么，也会调用super函数，最终会到达Root，Root则将充当接收器以结束super的调用链。

如何确保传递给super函数的参数有意义

  • 方法1：使用固定的参数位置，使用完全相同的参数

似乎是一个好的选择，但也有局限性，对于一个完整的类层级结构，我们不能指望它们在__init__()中采用相同的参数，在这种情况下，使用关键字参数剥离。

  • 方法二：关键字参数剥离，*args和**kwargs让你的函数接受任何位置参数和任何关键字参数，然后剥离你真正想要的关键字参数。

可以看到在第一个Sneaky类中，sneaky将被剥离，在Person中，human将被剥离，在object中，**kwargs中不会有任何东西。

*args和**kwargs主要用于函数定义，当我们需要定义的函数的传入参数个数不确定时，可以使用*args和**kwargs代替不确定的参数个数。
