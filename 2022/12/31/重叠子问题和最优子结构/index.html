重叠子问题和最优子结构

重叠子问题

众所周知，斐波那契数列的数学形式就是递归的

这样写代码虽然简洁易懂，但是十分低效，计算该递归的时间复杂度，首先计算子问题个数即二叉树节点总数为指数级别，所以子问题个数为 O(2^n)，每个子问题没有循环，只有 f(n - 1) + f(n - 2) 一个加法操作，时间为 O(1)，时间复杂度为 O(2^n)，观察递归树，很多节点存在大量重复计算，如何使用动态规划解决相同的子问题时的重复计算？

可以构建“备忘录”来记录计算过的节点

将memo独立出来成为一张表，通常叫做 DP table，在这张表上完成「自底向上」的推算

当前状态只和之前的两个状态有关，其实并不需要那么长的一个 DP table 来存储所有的状态，进一步优化，把空间复杂度降为 O(1)

一般来说是把一个二维的 DP table 压缩成一维，即把空间复杂度从 O(n^2) 压缩到 O(n)。

最优子结构

以找零问题为例，如何符合最优子结构，假设你有面值为 1, 2, 5 的硬币，你想求 amount = 11 时的最少硬币数（原问题），如果你知道凑出 amount = 10, 9, 6 的最少硬币数（子问题），你只需要把子问题的答案加一（再选一枚面值为 1, 2, 5 的硬币），求个最小值，就是原问题的答案。因为硬币的数量是没有限制的，所以子问题之间是互相独立的。

虽然思路没有问题，但是会超时，假设目标金额为n，给定的硬币个数为k，那么递归树最坏情况下高度为n（全用面额为1的硬币），再假设这是一棵满k叉树，则节点总数在 k^n 这个数量级。

同之前的例子，构造带备忘录的递归，稍作修改

很显然「备忘录」大大减小了子问题数目，完全消除了子问题的冗余

也可以自底向上使用 DP table 来消除重叠子问题，dp 数组的定义：当目标金额为 i 时，至少需要 dp[i] 枚硬币凑出。

初始化为amount+1相当于初始化为正无穷，因为硬币数最多可能等于amount，不直接初始化为Integer.MAX_VALUE是因为dp[i-coin]+1这样就会导致整型溢出。
