SQL注入

mysql 四自带库为：information_schema、mysql、performance_schema、sys

information_schema

TABLES：提供了关于数据库中的表的信息（包括视图），详细表述了某个表属于哪个 schema、表类型、表引擎、创建时间等信息，show tables from schema_name 的结果取之此表。

COLUMNS：提供了表中的列信息，详细表述了某张表的所有列以及每个列的信息，show columns from schema_name.table_name的结果取之此表。

table_schema：数据库的名称

table_name：具体的表名

table_type：表的类型

Less-1~4

是否存在注入点，同时确定闭合方式

?id=1'
语法错误，说明存在注入，单引号没有被过滤

原因是无论字符型还是数字型都会因为单引号个数不匹配而报错，如果未报错，不代表不存在 Sql 注入，因为有可能页面对单引号做了过滤，这时可以使用判断语句进行注入。

同时确定闭合方式是加单引号'

注入方式

?id=1' and '1'='1

查询成功

?id=1' and '1'='2

语法没有错误，但查询失败，说明是字符型注入

判断字段数

?id=1' order by 3--+

查询成功

?id=1' order by 4--+

查询失败，说明users表总字段数为3

确定回显位置

?id=-1' UNION SELECT 1,2,3--+

回显位置在2，3两位。

可以发现’‘1’‘) LIMIT 0,1’附近的语法错误，单引号包裹的’1’‘) LIMIT 0,1，这说明用’)闭合，也就是(‘$id’)

不存在回显位？

再用之前的注入方法已经不行了，因为页面不会将查询的信息返回。

可以用报错注入获取库名，表名，字段名，用到函数updatexml(目标xml内容,xml文档路径,更新内容)

  1. 该函数一般配合and或者or使用，和联合查询不同，不用在意字段数
  2. and情况下，只要前面一有false，后面不再被判断，同Less-1中的不同，所以建议使用or
  3. 报错一般长度有限制，不要太长，所以尽量别用group_concat()
  4. 因为报错注入只显示一条结果，所以需要使用limit语句
