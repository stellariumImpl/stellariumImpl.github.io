标准输入 输出 错误

默认情况下，每个进程有3个文件描述符

stdin标准输入，文件描述符0

stdout标准输出，文件描述符1，重定向>等同于1>，追加标准输出>>

stder标准错误，文件描述符2，重定向2>，将标准错误重定向到标准输出的位置2>&1，将标准输出和标准错误重定向到同一个文件> output 2>&1

< 输入重定向

>/dev/null 黑洞，任何写入到这里的数据都会被丢弃，比如2>/dev/null，标准错误被丢掉，命令行不会有任何输出。适用于不需要命令的输出，只需要执行这条命令

git

git暂存区（Staging Area）位于工作目录（Working Directory）和本地仓库（Local Repository）之间。暂存区允许你在提交之前整理和预览即将提交的更改，便于准备多步以及预览提交。

add: 将更改存储到暂存区（准备提交）

commit: 将暂存区的更改保存到本地仓库

push: 将本地commit过的内容发送到远程仓库

merge: 将一个分支的更改合并到另一个分支

clone: 将远程仓库的内容，完整复制下来到本地

pull: 从远程仓库获取到内容，合并到本地环境里

basename

用于去除文件路径，并且可以去掉文件的扩展名部分，比如basename helloworld.sh .jpg表示如果helloworld.sh时jpg文件，则去掉后缀名只输出前缀部分。

convert

用于图像格式转换处理，比如convert "$jpgFile" "$pngFile"，"$pngFile"指定了图像文件名称及格式。加上if，比如convert "$jpgFile" "$pngFile"，在运行convert命令的同时检查是否转换成功。

-e

if [ -e "$pngFile" ]; then 检查文件是否存在，-e 是 test 命令的一个选项，用于检查文件是否存在（存在即为真）。由于 test 是命令，因此必须在它的两边留有空格，即 if [ -e "$pngFile" ]; then。
