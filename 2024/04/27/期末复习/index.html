Page Cost Model

Number of tuples: Relation, which is a set of tuples, contains r tuples.

Tuple size: R bytes of size

Number of pages: 就存储设备而言，一个table存储可以在n个文件上，每个文件上有b个pages。

Page size(tuples amount): up to c tuples

Page size(bytes): B bytes of size

Consider the SQL table definition:

Assume that:

  • integer values are 4 bytes long
  • real values are 8 bytes long
  • char(X) values occupy X bytes
  • each tuple has a 4-byte header before the data
  • pages are are B = 4KB in size
  • each page has a 96-byte header; the rest of the page holds tuples
  • there are r = 10000 tuples in the Tab table

Calculate the following:

  1. R = the size of each tuple (in bytes)
  2. c = the number of tuples in each page
  3. b = the total number of pages needed to store Tab tuples

State all assumptions. Show all working.

解答

总结

  • 求R: 审清题目已知信息，和SQL语句中的字段信息，累加可得
  • 求c: floor((B - page header size)/R)，即(Page size(bytes)-Page header size)/size of each tuple， 向下取整即可得到Page size(tuples amount)
  • 求b: ceil(r/c)，即Number of tuples/Page size(tuples amount)，向上取整即可得到Number of pages

How many records can fit in a page?

The answer is c, which represents capacity of records for each page.

影响因素

  • page size: 1KB, …
  • record size: 64B, …
  • page header data: 4B - 32B
  • slot directory: depends on how many records

We consider average record sizeR

Given c, Header Size + c * Slot Size + c*R <= Page Size

Exercise: Space Utilisation

Consider the following page/record information:

  • page size = 1KB1KB1KB = 102410241024 bytes = 2102^{10}210 bytes
  • records: (w:int,x:varchar(20),y:char(10),z:int)
  • records are all aligned on 4-byte boundaries
  • x field padded to ensure z starts on 4-byte boundary
  • each record has 4 field-offsets at start of record (each 1 byte)
  • char(10) field rounded up to 12-bytes to preserve alignment
  • maximum size of x values = 20 bytes; average size = 16 bytes
  • page has 32-bytes of header information, starting at byte 0
  • only insertions, no deletions or updates

Calculate c = average number of records per page.

解答

根据each record has 4 field-offsets at start of record (each 1 byte)，我们得知每个record都需要空出4-byte偏移量；

对于每个record，需要满足4-byte padding boundary，对于(w:int,x:varchar(20),y:char(10),z:int)，大小为4(int)+16(average size)+10(char)+2(padding to 32 bytes)+4(int)=36 bytes；

每个record总共大小也就是average record size占40 bytes；

Page SizeB为1024 bytes，但是其中包含32位的Header Size；

根据公式Header Size + c * Slot Size + c*R <= Page Size，我们可以假设lose c bytes of Freespace to directory，也就是Slot Size为1 byte，32 + c + c * 40 <= 1024，解得c最大为24。
