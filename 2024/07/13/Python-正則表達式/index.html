特殊字符

  • \d代表任意數字[0-9]，\D代表任何非數字[^0-9]

  • \w代表任意字符[a-zA-Z0-9]，\W代表任意非字符[^a-zA-Z0-9]

  • \s代表任意空白符[\t\n\r\f]，\S代表任意非空白符[^\t\n\r\f]

  • \b代表任意單詞邊界，\B代表非單詞邊界

  • \A代表字符串的開始，相當於^，\Z代表字符串的末尾，相當於$

Raw string與特殊字符的關係

爲了在正則表達式中正確使用這些轉義字符，統一使用Raw string來定義正則表達式

沒有r前綴的字符串 的 普通字符串，需要對反斜杠進行雙重轉義，因爲其中的反斜杠\會被解釋爲轉義字符，需要\\才能表示一個反斜杠，注意，是對轉義字符，對通配符和Raw string效果大差不差。

对通配符和特殊字符的影响

通配符（如 .、*、[] 等）：普通字符串和 Raw string 在这些通配符的表示上并没有区别，因为这些字符在字符串解析阶段就已经被转义或者不需要转义。

转义字符（如 \n、\t、\\ 等）：普通字符串中需要使用双反斜杠 \\ 表示字面上的反斜杠，而 Raw string 则直接使用单反斜杠 \，不需要额外的转义。

小結 1

  • 普通字符串（没有 r 前缀）：

在普通字符串中，反斜杠 \ 是一个转义字符，用于转义后面的字符。
如果要表示一个反斜杠字符本身，需要使用两个反斜杠 \\，因为第一个反斜杠会转义第二个反斜杠，使其变成一个字面上的反斜杠字符。

  • Raw String（有 r 前缀）：

Raw string 不会解释反斜杠 \ 作为转义字符，它会把反斜杠后面的字符当作字面上的字符对待。
在 Raw string 中，如果你写 \，它就是 \ 字符本身，不会转义后面的字符。
